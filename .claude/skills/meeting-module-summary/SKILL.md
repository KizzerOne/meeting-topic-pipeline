# 功能模块会议文档

用于为每个功能模块生成一个可持续更新的 Markdown 文件。

## 每个模块必须包含

1. 模块目标
2. 相关会议来源
3. 已确认事项
4. 待办事项
5. 风险或未解决问题
6. 最近变化

## 写作要求

- 使用中文 Markdown。
- 用表格承载待办、风险、结论等结构化信息。
- 不要写空话，不要重复整段原始会议内容。
- 负责人、时间、状态缺失时写“原文未明确”。
- 保持模块文档能被新人直接阅读和接手。
